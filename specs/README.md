本目录包含 vitamin-bun 项目的所有技术规范文档，采用 OpenSpec 风格组织。

## 规范索引

### 核心规范

- [000-overview.md](./000-overview.md) - 项目总览和愿景
- [001-architecture.md](./001-architecture.md) - 系统架构设计

## 规范编写指南

### 命名规范

规范文档使用三位数字前缀 + 描述性名称的格式：

- `000-099`: 核心概念和架构
- `100-199`: 核心包规范
- `200-299`: 扩展功能规范
- `300-399`: 工具和 CLI 规范
- `400-499`: 配置和约定规范
- `500-599`: API 设计规范
- `600-699`: 测试和质量保证规范
- `700-799`: 部署和运维规范
- `800-899`: 最佳实践和指南
- `900-999`: 参考资料和附录

### 文档结构

每个规范文档应包含以下部分：

1. **概述** - 简要说明规范的目的和范围
2. **动机** - 为什么需要这个规范
3. **详细设计** - 详细的技术设计和实现
4. **示例** - 使用示例和代码片段
5. **权衡** - 设计决策的权衡和考虑
6. **替代方案** - 考虑过但未采用的方案
7. **未来工作** - 后续可能的改进和扩展

## 如何贡献

如果您想要添加新的规范或修改现有规范，请：

3. 确保文档遵循上述结构和命名规范
4. 提交 Pull Request
